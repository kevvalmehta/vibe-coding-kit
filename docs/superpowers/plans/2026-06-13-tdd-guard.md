# TDD-Guard Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A deterministic `PreToolUse` hook that blocks implementation-code edits during a build unless a failing test already exists (test-first), with a refactor escape hatch.

**Architecture:** One Python module `scripts/tdd_guard.py` split into small pure functions (`classify`, `decide`, `read_marker`) plus two side-effecting wrappers (`tests_are_red` runs the suite; `main` parses stdin and emits the deny decision). The pure functions hold all logic and are unit-tested directly; the wrappers are tested with monkeypatched subprocess + stdin. A gitignored `.tdd-guard` marker file at repo root turns the guard on/off and selects the mode.

**Tech Stack:** Python 3 (stdlib only — `json`, `sys`, `subprocess`, `pathlib`), pytest. No third-party packages. No LLM.

---

## File Structure

- `scripts/tdd_guard.py` — the guard. Pure logic + thin I/O wrapper. One file, one responsibility.
- `tests/test_tdd_guard.py` — pytest unit tests for every pure function + the wrapper.
- `.gitignore` — add `.tdd-guard` so the marker never commits in the "on" state.
- `.claude/settings.json` — add the `PreToolUse` hook entry.
- `AGENTS.md`, `SKILL-MAP.md` — register the guard (Hard Rule #6 portability).

## Verified ground-truth (Claude Code hooks, checked 2026-06-13)

- stdin JSON top-level: `tool_name`, `tool_input`, `cwd`. File path is `tool_input.file_path` for Edit/Write/MultiEdit.
- **Block** = exit 0 + stdout JSON:
  `{"hookSpecificOutput":{"hookEventName":"PreToolUse","permissionDecision":"deny","permissionDecisionReason":"..."}}`
- **Allow** = exit 0 with NO stdout (defers to normal permission flow).
- Gotcha: nothing else may be printed to stdout. Only the JSON, only when denying.

```python
# scripts/tdd_guard.py
"""TDD-Guard: PreToolUse hook that enforces test-first edits during a build.

Off by default. Active only when a `.tdd-guard` marker file exists at repo root.
See docs/superpowers/specs/2026-06-13-tdd-guard-design.md.
"""

CODE_EXTS = {".py", ".pyi", ".ts", ".tsx", ".js", ".jsx", ".mjs", ".cjs"}


def classify(file_path: str) -> str:
    """Return 'test', 'impl', or 'noncode' for a target file path."""
    p = file_path.replace("\\", "/").lower()
    name = p.rsplit("/", 1)[-1]
    if (
        "/tests/" in f"/{p}"
        or name.startswith("test_")
        or name.endswith("_test.py")
        or name.endswith((".test.ts", ".spec.ts", ".test.js", ".spec.js"))
    ):
        return "test"
    ext = name[name.rfind("."):] if "." in name else ""
    return "impl" if ext in CODE_EXTS else "noncode"
```

```python
def decide(mode, classification: str, tests_red: bool) -> str:
    """Return 'allow' or 'block'.

    mode: None (guard off) | 'strict' | 'refactor' | anything else (treated as strict).
    """
    if mode is None:
        return "allow"
    if classification in ("test", "noncode"):
        return "allow"
    if mode == "refactor":
        return "allow"
    # strict (or unknown mode) + impl code:
    return "allow" if tests_red else "block"
```

```python
from pathlib import Path

MARKER = ".tdd-guard"
DEFAULT_TEST_CMD = "python -m pytest -q"


def read_marker(root: Path):
    """Return {'mode':..., 'test':...} if the marker exists at root, else None."""
    marker = Path(root) / MARKER
    if not marker.exists():
        return None
    cfg = {"mode": "strict", "test": DEFAULT_TEST_CMD}
    for line in marker.read_text(encoding="utf-8").splitlines():
        line = line.strip()
        if not line or line.startswith("#") or ":" not in line:
            continue
        key, value = line.split(":", 1)
        cfg[key.strip()] = value.strip()
    return cfg
```

```python
import subprocess


def tests_are_red(test_cmd: str, cwd) -> bool:
    """Run the test command; True if a test fails (non-zero exit)."""
    result = subprocess.run(
        test_cmd, shell=True, cwd=str(cwd),
        stdout=subprocess.DEVNULL, stderr=subprocess.DEVNULL,
    )
    return result.returncode != 0
```

```python
import json
import sys

BLOCK_MSG = (
    "TDD-Guard: all tests are green — write a failing test before adding or changing "
    "implementation code. (Refactoring? set 'mode: refactor' in .tdd-guard.)"
)


def main() -> int:
    raw = sys.stdin.read()
    data = json.loads(raw) if raw.strip() else {}

    if data.get("tool_name") not in ("Edit", "Write", "MultiEdit"):
        return 0  # not an edit; allow (defer)

    file_path = data.get("tool_input", {}).get("file_path", "")
    root = Path(data.get("cwd") or Path.cwd())

    marker = read_marker(root)
    if marker is None:
        return 0  # guard off

    classification = classify(file_path)
    tests_red = False
    if classification == "impl" and marker.get("mode") == "strict":
        tests_red = tests_are_red(marker.get("test", DEFAULT_TEST_CMD), root)

    if decide(marker.get("mode"), classification, tests_red) == "block":
        print(json.dumps({
            "hookSpecificOutput": {
                "hookEventName": "PreToolUse",
                "permissionDecision": "deny",
                "permissionDecisionReason": BLOCK_MSG,
            }
        }))
    return 0


if __name__ == "__main__":
    sys.exit(main())
```
